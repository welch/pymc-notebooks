Hot Hot Hands¶

An exercise in Kruschke's Doing Bayesian Data Analysis has you build a probability model for pairs of free throws, the second conditioned on the success of the first, and asks whether a "hot hand" phenomenon exists among NBA players.

There's a nice demo running PyMC on IPCluster to solve this problem for many players at once, but why should they have all the fun? Here we work the original Larry Bird hot hands exercise, with some other bonuses.

import pymc
import numpy as np
from pymc import deterministic

class Struct:
    # convenience 'bag of attributes' class
    def __init__(self, **entries): self.__dict__.update(entries)

def posterior_hist(trace, label="", HDI=.95, alpha=1.0):   
    """Render a histogram of the posterior sampling distribution of a model variable."""
    t = sorted(trace)
    h = hist(t, normed=True, alpha=alpha, label="%s:%.2f" % (label, mean(t)))
    ymax = max(h[0])
    axvline(mean(t), c='red', linestyle='-.')
    hdidx = int(len(t)*(1 - HDI)/2.) 
    annotate("", xy=(t[hdidx],0), xytext=(t[-hdidx],0),
             arrowprops=dict(arrowstyle="|-|", connectionstyle="arc3", color='k'))

The text presents Larry Bird's free throw data as shot totals after an initial make or miss. We need to synthesize a list of throw events (success/fail) that match these proportions, to serve as the observed data for our Bernoulli likelihoods:

In [132]: 
# free throw data for Larry Bird, 1980-1982. 
#
throws = Struct(
    hit_hit = [i < 251 for i in range(285)], # list of throw successes after an initial hit
    hit_miss = [i < 48 for i in range(53)]   # list of throw successes after an initial miss
)

On to the model. We are estimating two parameters: the probabilities of making a second throw, after making or missing the first throw. We are given the make/miss counts, which can be treated as Bernoulli trial outcomes. To perform Bayesian estimation of the two probabilities, Kruschke suggests a Beta(30,10) prior for each, corresponding to the belief that NBA players make about 75% of their throws, \(E[[Beta_{\alpha,\beta}]] = \alpha/(\alpha+\beta)\).

In [133]: 
# Beta priors for our Bernoulli likelihoods. 
# Assume NBA players make 75% of their shots.
priors = Struct(
    hit_hit =  pymc.Beta('phit_hit',  30, 10),# prob. of making second after making first
    hit_miss = pymc.Beta('phit_miss', 30, 10) # prob. of making second after missing first
)

# likelihood for observed second throws. Each throw is a Bernoulli trial.
likely = Struct(
    hit_hit = [pymc.Bernoulli('hit_hit%d' % i, p=priors.hit_hit, value=throw, observed=True) 
               for i, throw in enumerate(throws.hit_hit)],
    hit_miss = [pymc.Bernoulli('hit_miss%d' % i, p=priors.hit_miss, value=throw, observed=True)
                for i, throw in enumerate(throws.hit_miss)]
)

# The statistic we care about is the difference between the hit/miss probabilities
@deterministic
def diff(phit_hit=priors.hit_hit, phit_miss=priors.hit_miss):
    return phit_hit - phit_miss

# Instantiate and fit the model
S = pymc.MCMC([priors.hit_hit, priors.hit_miss, likely.hit_hit, likely.hit_miss, diff], db='pickle')
S.sample(10000, burn=1000, thin=2)
posterior_hist(S.trace('phit_hit')[:], "hit_hit", alpha=.5)
posterior_hist(S.trace('phit_miss')[:], "hit_miss", alpha=.5)
legend()
show()
posterior_hist(S.trace('diff')[:], "diff", alpha=.5)
legend()
show()

Notice that a difference of 0 is within the 95% HDI, and thus it is credible to conclude no hot hand for Mr. Bird. But I'm bothered by using the same prior for hit/miss data having different numbers of observed hits and misses. To see this more clearly, consider a reparameterization of Beta in terms of its mean \(\mu\) and a "confidence" K:

\(Beta(a, b) = Beta(\mu*K, (1-\mu)*K)\)

K encodes the number of observations baked into the prior and controls its weight against new observations during updating. Since we use the same prior for both throw probabilities, we have built-in a bias towards these probabilities being the same. But because there are very different numbers of hit_hit vs hit_miss events, their posteriors show the effect of an equivalent K differently. The hit_miss prior is weighted almost equally to the number of observations during updating. Here is the model with a K instead scaled to the same fraction of the data size for each prior:

# Beta priors for our Bernoulli likelihoods. 
# We reparameterize Beta in terms of mean (mu) and confidence (K):
# Beta(a, b) = Beta(mu*K, (1-mu)*K). Assume NBA players make 75% of their shots.
mu, K_hit, K_miss = (.75, len(throws.hit_hit)/7., len(throws.hit_miss)/7., )
priors = Struct(
    hit_hit = pymc.Beta('phit_hit',mu*K_hit, (1-mu)*K_hit), # prob. of making second after making first

Wha? The miss density is centered farther right than the hit density? Consider the original raw data: 48/53 is indeed greater than 251/285. The only reason our initial estimate showed a slightly hot hand is the unequal biasing towards the prior because of data size!

Arguments for/against using same prior confidence (for: 75% is the marginal rate, assumes no effect. against: marginal rate says hit_hit will always be overrepresented in the data set).

Continuing this example, recall that if you want to count successes in N Bernoulli(p) trials (like we do here for throws), that count will be distributed as Binomial(n,p). If we specify a Binomial likelihood for throw counts instead of a product of N Bernoulli likelihoods for individual throws, we should get the same estimate.

# Beta priors for our Binomial likelihoods. 
# We reparameterize Beta in terms of mean (mu) and confidence (K):
# Assume NBA players make 75% of their shots.
# Because we now have only one observation for each parameter, we scale down K
# so the prior does not overshadow the data.
mu, K_hit, K_miss = (.75, 1., 1.)
priors = Struct(
    hit_hit = pymc.Beta('phit_hit',mu*K_hit, (1-mu)*K_hit), # prob. of making second after making first
    hit_miss = pymc.Beta('phit_miss', mu*K_miss, (1-mu)*K_miss) # prob. of making second after missing first
)
# likelihood for observed number of second throw successs. Each is ~ Binomial 
likely = Struct(
    hit_hit = pymc.Binomial('hit_hit', n = len(throws.hit_hit), p=priors.hit_hit, value=sum(throws.hit_hit), observed=True),
    hit_miss = pymc.Binomial('hit_miss', n = len(throws.hit_miss), p=priors.hit_miss, value=sum(throws.hit_miss), observed=True) 
)

@deterministic()
def diff(phit_hit=priors.hit_hit, phit_miss=priors.hit_miss):
    return phit_hit - phit_miss

# Instantiate and fit the model. Priors are same as earlier Bernoulli likelihood model.
S = pymc.MCMC([priors.hit_hit, priors.hit_miss, likely.hit_hit, likely.hit_miss, diff], db='pickle')
S.sample(10000, burn=1000, thin=2)

posterior_hist(S.trace('phit_hit')[:], "hit_hit", alpha=.5)
posterior_hist(S.trace('phit_miss')[:], "hit_miss", alpha=.5)
legend()
show()
posterior_hist(S.trace('diff')[:], "diff", alpha=.5)
legend()
show()

Continuing this example, why guess at K, when we can estimate it along with the other parameters? This will be a hierarchical model, and we'll need to specify a prior for K. We need a prior in the range \([1,\infty)\), and we'd like to bias it towards lower values (so the data has greater relative weight than the prior). Kruschke p433 introduces a nice transform of a Uniform[0,1] prior to the range \([1,\infty)\) for inferring a t-distribution's df parameter that is near-linear at low values. A gain parameter sets the slope of this curve for low values. This transformation also works well for our K. The Lambda instances below are one-liners that do the same job as the @deterministic() decorator.

If we were ingesting data for multiple players, it would make sense to estimate the population \(\mu\) as well. Instead we leave \(\mu\) as a constant derived from the entire NBA population, not something to be inferred from one player's stats.

In [142]: 
mu = .75
udf_gain = 10
priors = Struct()
priors.udf_hit = pymc.Uniform('udf_hit', 0, 1)
priors.udf_miss = pymc.Uniform('udf_miss', 0, 1)
priors.K_hit = pymc.Lambda('K_hit', lambda x=priors.udf_hit: 1 - udf_gain * np.log(1-x))
priors.K_miss = pymc.Lambda('K_miss', lambda x=priors.udf_miss: 1 - udf_gain * np.log(1-x))
priors.hit_hit = pymc.Beta('phit_hit',alpha=(mu * priors.K_hit), beta=((1 - mu) * priors.K_hit)) # prob. of making second after making first
priors.hit_miss = pymc.Beta('phit_miss', alpha=(mu * priors.K_miss), beta=((1 - mu) * priors.K_miss)) # prob. of making second after missing first

likely = Struct(
    hit_hit = [pymc.Bernoulli('hit_hit%d' % i, p=priors.hit_hit, value=throw, observed=True) 
               for i, throw in enumerate(throws.hit_hit)],
    hit_miss = [pymc.Bernoulli('hit_miss%d' % i, p=priors.hit_miss, value=throw, observed=True)
                for i, throw in enumerate(throws.hit_miss)]
)

@deterministic()
def diff(phit_hit=priors.hit_hit, phit_miss=priors.hit_miss):
    return phit_hit - phit_miss

# Instantiate and fit the model. 
S = pymc.MCMC([priors.udf_hit, priors.udf_miss, priors.K_hit, priors.K_miss, priors.hit_hit, priors.hit_miss, 
               likely.hit_hit, likely.hit_miss, diff], db='pickle')
S.sample(10000, burn=1000, thin=2)
posterior_hist(S.trace('K_hit')[:], "K_hit", alpha=.5)
posterior_hist(S.trace('K_miss')[:], "K_miss", alpha=.5)
legend()
show()
posterior_hist(S.trace('phit_hit')[:], "hit_hit", alpha=.5)
posterior_hist(S.trace('phit_miss')[:], "hit_miss", alpha=.5)
legend()
show()
posterior_hist(S.trace('diff')[:], "diff", alpha=.5)
legend()
show()

Varying udf_gain below 50 yields low K values, K_hit and K_miss similar in value. Turning the gain up to 1000 leads to a very high K_miss (and much lower K_hit), inverting the ultimate hot hand estimate. I don't want to think about the right way to choose udf_gain. It's turtles all the way down.
